## Блок A. Качество генерации и языка (пп. 1, 3, 4)

### П.1 — Некорректный белорусский язык в генерации 🔴
**Где:** [generator.js:435](../src/generator.js#L435) — жёстко зашито «Пиши по-русски» во всех генерациях; [generator.js:559](../src/generator.js#L559) (freeform) и [analyzer.js:96-102](../src/analyzer.js#L96-L102) — то же.
**Корень проблемы:** язык генерации всегда русский, а предмет «Белорусский язык» требует вывода на белорусском. Плюс базовая модель (`gpt-5-mini` / `qwen3-32b`, [generator.js:423](../src/generator.js#L423)) слабо владеет белорусским → «выдуманный» язык.
**Что делать:**
1. Ввести определение языка вывода по предмету: если `student.subject` содержит «белорус»/«бел. яз»/«мова» → язык вывода = белорусский, иначе русский. Вынести в хелпер `outputLanguage(student)`.
2. Заменить хардкод «Пиши по-русски» на подстановку языка во всех трёх местах (generate, solutions, analyzer).
3. **Ключевое:** оценить смену модели для белорусского. `gpt-5-mini`/`qwen3-32b` дают «кашу». Кандидаты с лучшим белорусским: более крупная модель (gpt-5, gemini-2.5-flash уже есть в прайс-листе [analyzer.js:5-8](../src/analyzer.js#L5-L8)). Нужен A/B-тест на реальных темах.
4. Добавить в промпт явное требование: «Пиши строго на литературном белорусском языке, не смешивай с русским; если не уверен в слове — используй нейтральную формулировку».
**Риск:** без смены модели пункты правки промпта дадут лишь частичный эффект. Требуется тест до релиза.
**Оценка:** промпт+язык — 🟡; выбор/тест модели — 🔴 (отдельная сессия).

### П.3 — Материал не соответствует классу/программе (напр. ускорение в 7 классе) 🟡

**Оценка:** 🟡.

### П.4 — Задачи не соответствуют белорусским учебникам/сборникам, не годятся для ЦТ/ЦЭ 🔴
**Где:** генерация «с нуля», без опоры на источники. Уже есть `searchMaterials` ([message-handler.js:211](../src/telegram/message-handler.js#L211)) и категория `ct_ce` ([telegram.js:205](../src/telegram.js#L205)).
**Что делать (варианты по возрастанию усилий):**
1. Дешёвый: добавить в промпт стиль/уровень белорусских сборников и уровень ЦТ/ЦЭ как отдельный режим сложности.
2. Средний: RAG — при генерации подтягивать релевантные задачи из базы материалов (`searchMaterials`) и просить модель опираться/переформулировать по образцу.
3. Дорогой (то, что просит пользователь): собрать БД задач из белорусских учебников/сборников и генерировать на её основе.
**Рекомендация:** начать с варианта 2 (RAG поверх уже существующей базы материалов) — переиспользует инфраструктуру.
**Оценка:** 🔴 (архитектурно, отдельная сессия).

## Блок D. Список учеников (пп. 8, 9, 10, 12)

### П.8 — В списке ученики только с пробным занятием 🟡
**Где:** два пути синка:
- [moy-klass.js:198-229](../src/moy-klass.js#L198-L229) (`sync`, админский `/api/moy-klass/sync`) — **НЕ применяет** `isAllowedClientState` и `visit`-фильтр, линкует всех из `joins`. **Баг.**
- [moy-klass.js:251-331](../src/moy-klass.js#L251-L331) (`syncTeacherStudents`, кнопка «Обновить») — фильтр есть (`visit !== true`, `isAllowedClientState`).
**Что делать:**
1. Применить те же фильтры статуса в `sync()` либо признать его устаревшим и убрать.
2. Проверить полноту `ALLOWED_CLIENT_STATE_IDS` ([moy-klass.js:10](../src/moy-klass.js#L10)) — возможно, статус «пробное проведено» имеет отдельный ID, который сейчас случайно проходит. Уточнить справочник clientStatuses в Мой Класс.
3. Ручное удаление (см. п.9) закроет остаточные случаи.
**Оценка:** 🟡.

### П.9 — Нет кнопок «Добавить/Удалить ученика» 🔴
**Где:** нет ни UI, ни методов. Есть `linkStudent`/`unlinkStudentByMyKlassId`/`pruneStudents` ([repositories.js:100-132](../src/database/repositories.js#L100-L132)), но нет ручного unlink по studentId и нет добавления вручную.
**Что делать:**
1. Репозиторий: добавить `unlinkStudent(tutorId, studentId)` (ручное удаление по id) и метод создания ученика вручную (`full_name`, `subject`, `grade`) с линковкой к тьютору. Учесть, что таблица `students` шарится с легаси-приложением (NOT NULL `name`, [repositories.js:88-95](../src/database/repositories.js#L88-L95)).
2. UI: в меню ученика — кнопка «🗑 Удалить из списка» (с подтверждением). В списке учеников — «➕ Добавить ученика» (сессия ввода имя/предмет/класс).
3. Важно: удаление должно быть «отвязкой от тьютора» (unlink), а не удалением из общей БД — ученик может принадлежать другому тьютору. Плюс: при следующем автосинке отвязанный ученик может вернуться — нужен флаг «скрыт вручную», иначе кнопка бесполезна.
**Оценка:** 🔴 (репо + UI + флаг подавления автосинка).

### П.10 — Неверно указан класс у части учеников 🟡
**Где:** класс определяется эвристикой [moy-klass.js:32-46](../src/moy-klass.js#L32-L46) (`gradeFromName`/`gradeFromUser`) из названия группы/атрибута. Хранится в `tutor_students.grade` ([repositories.js:100-106](../src/database/repositories.js#L100-L106)).
**Что делать:**
1. Дать ручное редактирование класса в карточке ученика (сейчас редактируется только `student_cards`, не `tutor_students.grade`). Нужен метод `updateStudentGrade(tutorId, studentId, grade)` + пункт в меню редактирования.
2. Автосинк не должен затирать вручную выставленный класс: `linkStudent` использует `COALESCE(EXCLUDED.grade, ...)` — не перезатрёт `null`, но перезатрёт вручную выставленное, если снова придёт значение из Мой Класс. Нужен флаг «класс задан вручную».
**Оценка:** 🟡 (UI + метод + защита от перезаписи синком).
